# FRAME GOVERNANCE

Repository standards and consistency rules.

## Rule Supremacy In case of any ambiguity or conflict between these standards and FRAME+CORE, the PRIORITY ORDER and rules defined in FRAME+CORE strictly override this document. 

---

# FILE NAMING RULES

Use deterministic naming for applicable documents.

---

# DOCUMENTATION RULES

Documentation must be:
- concise
- implementation-ready
- AI-readable
- human-readable
- deterministic

Avoid:
- vague language
- unnecessary theory
- duplicated explanations

---

# CODE STANDARDS

Required:
- type safety
- input validation
- error handling
- avoid magic values

Avoid:
- tightly coupled systems
- hidden side effects
- duplicated business logic
- magic values
- inconsistent naming

---

# DESIGN CONSISTENCY

UI systems must:
- follow shared design tokens
- preserve spacing consistency
- preserve typography consistency
- preserve interaction consistency
- support accessibility

---

# REFACTORING RULES

Prefer:
- incremental refactors
- isolated changes
- backward compatibility

Avoid:
- large rewrites
- unnecessary migrations
- architecture churn

# TASK MANAGEMENT

Tasks must contain:
- objective
- status
- dependencies
- acceptance criteria

Statuses:
- TODO
- IN_PROGRESS
- BLOCKED
- DONE
